# ⚡️ KABOOM!

> **BAM! Deploy snippets, containers & functions. No ops required.**

**Kaboom!** is a high-performance, visually striking Serverless Computing Deployment Engine. Wrapped in a premium "Comic Hyper-Pop" brutalist aesthetic, it provides developers with a frictionless platform to deploy, execute, and monitor backend code instantly.

---

## 🚀 Features

### 1. The Workbench (Deployment Matrix)
Kaboom! supports three distinct execution models:
- **Snippets**: Write or paste pure JavaScript/TypeScript directly into the integrated Monaco Editor. Natively supports both CommonJS and modern ECMAScript Modules (ESM) via dynamic Data URI imports.
- **Functions (.ZIP)**: Upload complete Node.js architectures containing `package.json`, `index.js`, and dependencies. Kaboom dynamically figures out your entry point and serves it.
- **Containers**: Drop in any pre-built Docker image from Docker Hub (e.g., `nginx:alpine`). Kaboom maps the internal container port to a live accessible endpoint on your machine.
- **GitHub Import**: Paste any raw GitHub code link (like a gist or repository script) into the editor to instantly pull it into your workspace.

### 2. The Lair (Observability Hub)
Once a gadget is zapped into reality, it enters **The Lair**:
- **Real-Time Telemetry**: Sparkline charts natively track the average latency of your function over its last active executions.
- **Live Terminal Streams**: View real-time output and crash diagnostics from `stdout` and `stderr` generated by your serverless runtimes.
- **Stress-Testing Engine**: Barrage your endpoints directly from the UI to test throughput and efficiency (Success/Fail metrics) under pressure.
- **Semantic Status Monitoring**: Every gadget dynamically adjusts its status depending on health and executes securely managed state updates.

### 3. Hyper-Pop Developer UX
Kaboom avoids the boring, flat SaaS look:
- Halftone backgrounds and comic-grid textures.
- A seamless transition system between light mode and a high-contrast true Dark Mode.
- Immersive, theme-aware semantic feedback mechanisms.

## 🛠️ Architecture Deep Dive

### Next.js 14 App Router
Built heavily around Server Actions and modern routing concepts to provide lightning-fast navigation with Framer Motion page transitions.

### Poly-Runtime Engine
- **`vm` Sandboxing**: Instantly evaluates simple code securely without touching the file system.
- **Child Process Isolation**: Zipped node functions are spun up in child processes via `execFile`, utilizing isolated `extractPath` buffers so your main Kaboom runtime is protected from crashes.
- **Hierarchical ESM Discovery**: For ZIP files, the invoking engine dynamically climbs export objects and fallback chains (`run`, `main`, `execute`, `handler`, `default`) to parse complex CJS/ESM interops flawlessly.

### Local Database
Uses **Prisma ORM** with **SQLite** to manage Gadget metadata (`publicUrl`, `endpoint`, configurations, etc.) and execution logs efficiently.

## 💻 Getting Started

### Prerequisites
- Node.js (v18+ recommended)
- Docker Desktop (Required if deploying Container Gadgets)

### Installation
1. Clone the repository:
   ```bash
   git clone https://github.com/your-username/kaboom.git
   cd kaboom
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Initialize the database:
   ```bash
   npx prisma db push
   ```
4. Start the reactor:
   ```bash
   npm run dev
   ```

Open [http://localhost:3000](http://localhost:3000) and **ZAP IT!**
